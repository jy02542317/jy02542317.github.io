类型、存储和变量

4.1 C#程序是一组类型声明

C#程序或DLL的源代码是一组类型声明

对于可执行程序，类型声明中必须有一个包含Main方法的类

命名空间是一种将相关的类型声明分组并命名的方法。

4.2 类型是一种模板

类型由下面的元素定义：

名称

用于保存数据成员的数据结构

一些行为及约束条件

4.3 实例化类型

通过实例化类型而创建的对象被称为类型的对象或类型的实例。

在C#程序中，每个数据项都是某种类型的实例。

4.4 数据成员和函数成员

数据成员：保存了与这个类的对象或整个类相关的数据

函数成员：执行代码。函数成员定义类型的行为。

4.5 预定义类型

13种简单类型和3种非简单类型

11种数据类型

  • 不同长度的有符号和无符号整数类型。
  • 浮点数类型float和double.
  • 一种称为decimal的高精度小数类型。

一种Unicode字符类型char

一种布尔类型bool。

3种非简单类型如下。

string，Unicode字符数组

object, 它是所有其他类型的基类

dynamic，使用动态语言编写的程序集时使用

4.6 用户定义类型

类类型 class

结构类型 struct

数组类型 array

枚举类型 enum

委托类型 delegate

接口类型 interface

4.7 栈和堆

4.7.1 栈

栈是一个内存数组，是一个LIFO（Last-In First-Out)的数据结构。

栈存储几种类型的数据

某些类型变量的值

程序当前的执行环境

传递给方法的参数

栈的特征

数据只能从栈的顶端插入和删除。

把数据放到栈顶称为入栈。

从栈顶删除数据称为出栈。

4.7.2 堆

堆是一块内存区域。在堆里可以分配大块的内存用于存储某种类型的数据对象。

4.8 值类型和引用类型

值类型只需要一段单独的内存，用于存储实际的数据。

引用类型需要两段内存。

第一段存储实际的数据，它总是位于堆中。

第二段是一个引用，指向数据在堆中的存放位置。

4.8.1 存储引用类型对象的成员

引用类型对象的数据部分始终存放在堆里。

值类型对象，或引用类型数据的引用部分可以存放在堆里，也可以存放在栈里，这依赖于实际环境。

4.8.2 C#类型的分类

值类型： sbyte, byte, short, ushort, int, uint, long, ulong, bool, struct, enum

引用类型：float, object,double,char,decimal,object,string,dynamic,class,interface,delegate,Array
